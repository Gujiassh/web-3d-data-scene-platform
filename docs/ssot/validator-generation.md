# Validator Generation

> Status: Accepted
> Updated: 2026-07-22

SceneWeave commits AJV standalone validators under `packages/document/src/generated`. Typecheck, build, package
verification, and explicit contract generation may run while the Studio Vite server is reading those modules.

## Write Contract

- If generated bytes are unchanged, generation must not rewrite, rename, or update the file timestamp.
- If generated bytes change, generation must write and sync a temporary file in the target directory, then atomically
  rename it over the target.
- If replacement fails, the previous complete target remains available and the temporary file is removed.
- Generated validator bytes, ESM default exports, schema validation behavior, and package exports remain unchanged.

Directly calling `writeFile` on a live generated module is forbidden. It truncates the target before writing the new
bytes, so Vite can observe an empty module and cache that transform under a timestamped URL. The resulting browser error
claims that the validator has no default export even though the completed file does export one.

## Verification

- Unit tests cover unchanged generation, atomic replacement, and failed replacement cleanup.
- The standalone validator smoke test must still import every generated ESM default export and validate current and
  migrated fixtures.
- Runtime acceptance must fetch the formerly failing validator URL and open the Studio without module export or runtime
  errors while generation runs.
