# Smart-Home Starter Generator

This generator freezes the owner-provided `/mnt/e/data/model/smart_home_90sqm` snapshot into an ordinary
SceneDocument 1.4 archive. It does not extend the document, repository, or archive contracts.

## Local Validation

```bash
node scripts/smart-home/generate.mjs \
  --mode local-validation \
  --output /home/cc/tmp/web3d-smart-home-starter
```

Local validation may run without a redistribution authorization, but its output must resolve below `/home/cc/tmp`.
The generated report records `licenseGate.status=blocked`; these bytes must not be committed, deployed, or released.

For a determinism comparison, generate into two new local paths and compare all three outputs:

```bash
node scripts/smart-home/generate.mjs --mode local-validation --output /home/cc/tmp/web3d-smart-home-a
node scripts/smart-home/generate.mjs --mode local-validation --output /home/cc/tmp/web3d-smart-home-b
sha256sum /home/cc/tmp/web3d-smart-home-{a,b}/*
```

For local clean-profile Chromium evidence, start Studio and run the controller against one generated output:

```bash
pnpm --filter @web3d/studio dev --host 127.0.0.1
node scripts/smart-home/browser-evidence.mjs \
  --archive /home/cc/tmp/web3d-smart-home-a/smart-home-90sqm.web3d.zip \
  --descriptor /home/cc/tmp/web3d-smart-home-a/descriptor.json \
  --output /home/cc/tmp/web3d-smart-home-browser-evidence
```

The controller uses a fresh Chromium context, intercepts only the local descriptor/archive requests, and captures the
deterministic Run snapshot that simultaneously shows `ready`, application-level `offline` and `alarm` values. It writes
a full-page screenshot plus `evidence.json` with browser identity, DOM runtime state, alarm levels, Canvas hash and color
count. Both generated bytes and evidence must remain below `/home/cc/tmp`; this local evidence path is intentionally not
part of CI.

## Public Generation Gate

Public generation requires `--mode public --license-record <absolute-path>`. The authorization JSON must contain:

```json
{
  "schemaVersion": "1.0.0",
  "redistributionAuthorized": true,
  "copyrightOwner": "Exact copyright owner",
  "licenseId": "SPDX-or-explicit-license-id",
  "licenseTextPath": "LICENSE.txt",
  "licenseTextSha256": "64 lowercase hex characters",
  "coveredAssetSha256": ["every exact hash from allowlist.mjs"]
}
```

The license text path is resolved relative to the authorization record. Public generation fails unless the text hash
matches and the covered hash set equals all 38 selected SHIP assets. No authorization record is tracked while ownership
and redistribution terms remain unconfirmed.

## Output Contract

The generator emits `smart-home-90sqm.web3d.zip`, `descriptor.json`, and `generation-report.json`. The bootstrap-facing
descriptor is intentionally small and validated by `fixtures/starter-descriptor.schema.json`:

```json
{
  "schemaVersion": "1.0.0",
  "projectId": "smart-home-90sqm",
  "archiveUrl": "/starter/smart-home-90sqm.web3d.zip",
  "archiveSha256": "<sha256>",
  "archiveByteLength": 1
}
```

Generation hard-validates the registry/status/floorplan snapshot, the exact 38-entry external SHIP allowlist, manifest
verdict, GLB hash and byte length, declared triangles, zero glTF validator errors, Three.js load verdict, and fixed
semantic node indices. `smart_toilet` remains explicitly excluded because its manifest verdict is `NO-SHIP`.
`recessed_downlight` is also excluded while its source status is `IN_PROGRESS` and its manifest remains `NO-SHIP`
with failed validator/Three.js gates.

## Presentation Shell

The archive contains one additional code-generated `presentation_shell` SceneAsset under CC0-1.0. It is generated from
the hash-bound owner source `00_specs/floorplan.json` into a Y-up GLB with eight low-thickness colored room floors and all
27 exterior and interior wall segments represented as 0.22-0.28 m low walls. It has no ceiling or top panel. The source
floorplan is read only from `--source-root` and is not copied into this repository. The report records its exact hash,
plus the tracked CC0 text, generator path, GLB hash, byte length and Runtime-facing stats.

The presentation shell is a locked visible root entity and disables the scene grid. The owner-provided `home_shell`
remains in the document as a locked hidden entity. Independent entrance/balcony/interior doors, casement windows and the
three motorized curtains remain editable in the tree but start hidden so they do not obscure the low-wall presentation.
All other furniture and smart devices start visible.

The external redistribution authorization remains hash-bound to exactly the original 38 SHIP assets. The generated CC0
shell is reported separately and never enters `coveredAssetSha256`; local validation therefore remains blocked for public
distribution until those 38 owner-provided assets receive explicit authorization.
